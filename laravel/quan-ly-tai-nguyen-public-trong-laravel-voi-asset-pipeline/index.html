    Quản lý tài nguyên public trong Laravel 4 với Asset Pipeline

    Cách thức quản lý các tài nguyên public gồm css files, js files, images,...
    04 Oct 2014 in Laravel  10.79 minutes read

    Trong laravel 4, có nhiều cách để bạn quản lý các tài nguyên css, js, hình ảnh của mình. Trong bài này, tôi sẽ giới thiệu với bạn một cách để quản lý các tài nguyên (assets) để chúng có thể phục vụ trình duyệt một cách hiệu quả nhất có thể với Asset Pipeline.

    Laravel 4 không có sẵn tính năng này. Nhưng bạn hoàn toàn có thể cài đặt một package từ cộng đồng để có được chức năng tuyệt vời này trong project của bạn.

    Note: Asset Pipeline có thể hiểu là đường ống dẫn tài nguyên. Trong khuôn khổ bài viết, tôi sẽ sử dụng từ Asset Pipeline để bạn có thể quen với từ này và tìm kiếm tài liệu cũng như kiến thức liên quan bằng tiếng Anh được dễ hơn.

    What is an asset pipeline?

    Trước tiên, có lẽ tôi nên giải thích chính xác thì nó là cái gì.

    Khi triển khai một website trên Internet, trình duyệt phải lấy tất cả các assets (bao gồm các file css, js, images) để tạo nên trang web. Do đó, để lấy được mỗi tập tin CSS hay Javascript, trình duyệt cần tạo ra các request riêng. Theo thời gian, các file CSS và Javascript có thể sẽ lớn dần lên, nếu bạn làm với các project lớn.

    Asset pipeline sẽ tự động kết hợp tất cả các files tài nguyên vào trong một file duy nhất và tối thiểu hóa (minify) code trong đó. Điều đó có nghĩa là trình duyệt chỉ phải thực hiện 1 request duy nhất, kích cỡ file cũng giảm đi đáng kể và nó sẽ có ảnh hưởng lớn đến hiệu năng của trang web.

    Thời gian gần đây, các developers có xu hướng viết CSS sử dụng LESS hoặc SASS, hoặc họ dùng CoffeeScript thay vì Javascript thông thường. Asset pipeline cũng sẽ tự động biên dịch tất cả chúng thành một file CSS hoặc Javascript thông thường.

    Thêm nữa là, asset pipeline cũng sẽ tự động cache các files stylesheets và javascripts khi bạn triển khai trong môi trường production. Do đó, các files này sẽ không phải download lại mỗi lần tải lại trang nếu chúng không có gì thay đổi.

    Đó là các lý do tại sao bạn nên sử dụng asset pipeline. Asset pipeline có rất nhiều lợi ích tuyệt vời, ngay cả khi bạn không có ý định sử dụng một bộ tiền xử lý (preprocessor) (như LESS hay SASS) hay chỉ muốn viết javascript thuần :).

    Install Asset Pipeline

    Hiện tại, tôi đang sử dụng package Asset Pipeline này trong các project của mình.

    Để cài đặt package, đơn giản là bạn chỉ cần thêm dòng sau dưới đây vào file composer.json:

    "require": {
  "codesleeve/asset-pipeline": "dev-master"
}

    Tiếp đó, bạn chạy lệnh sau để cập nhật các phụ thuộc và tải package asset pineline về:

    composer update

    Tiếp đến, chúng ta cần nói cho Laravel biết về service provider (người cung cấp dịch vụ). Bạn thêm dòng code dưới dây vào phần providers trong file app/config/app.php:

    'Codesleeve\AssetPipeline\AssetPipelineServiceProvider'

    Sau đó, asset pipeline sẽ có các cách hành xử khác nhau phụ thuộc vào môi trường bạn đang thiết lập. Trong môi trường production, tất cả các file của bạn sẽ được nén lại (compress), ghép nối với nhau (concatenate) và sao lưu (cache). Nếu bạn không muốn điều đó xảy ra trong môi trường developing, bạn cần thiết lập chính xác môi trường triển khai của mình. Để chắc chắn rằng bạn đang thiết lập môi trường local cho project của mình, bạn kiểm tra phần thiết lập môi trường trong file bootstrap/start.php như sau:

    $env = $app->detectEnvironment(array(
	'local' => array('localhost'),
));

    Cuối cùng, bạn cần tạo ra các files asset của riêng mình bằng cách nhập vào lệnh sau từ Terminal: 

    $ php artisan assets:setup

    Working with asset pipeline

    Và giờ, project của bạn đã được cài đặt asset pipeline. Bạn sẽ thấy có thêm vài thư mục mới trong project. Đó là thư mục assets ở dưới thư mục app/, và thư mục provider/ ngang hàng với thư mục app/:

    /provider/assets/javascripts/jquery.min.js
/provider/assets/stylesheets/readme.txt
/app/assets/javascripts/application.js
/app/assets/images/readme.txt
/app/assets/stylesheets/application.css

    Hai files /app/assets/javascripts/application.js và /app/assets/stylesheets/application.css sẽ bao gồm tất cả các files tài nguyên khác trong cùng thư mục và chúng ta sử dụng chúng để cung cấp các tài nguyên cho trang web.

    Để thêm các asset files vào trong HTML template, bạn thêm hai thẻ sau:

    <?= stylesheet_link_tag() ?>
<?= javascript_include_tag() ?>

    Hai thẻ trên sẽ generate ra chính xác thẻ link và script khi bạn chạy trên trình duyệt:

    <link rel="stylesheet" type="text/css" href="http://localhost:8000/assets/application.css">
<script src="http://localhost:8000/assets/application.js"></script>

    Bạn có thể thêm hình ảnh vào views một cách dễ dàng với thẻ image_tag(). Giả sử, bạn có hình ảnh về chú chim Tweety đặt trong file app/assets/images/tweety.png, bạn sẽ thay thế hình ảnh icon Laravel bằng hình ảnh Tweety Bird. Bạn thay thế nội dung trong phần body của file app/views/hello.php thành: 

    <div class="welcome">
	<?= image_tag('tweety.png', ['alt' => 'Tweety']) ?>
<h1>You have arrived.</h1>
</div>

    Bạn mở trình duyệt và gõ vào địa chỉ http://localhost:8080 (tùy thuộc vào cổng khi bạn chạy lệnh php artisan serve nhé :D) và Enter, kết quả sẽ như thế này:

    Hình ảnh chú Tweety Bird của bạn sẽ được tải lên với đường dẫn http://localhost:8000/assets/tweety.png. Bạn có thể sử dụng Bug Tool để xác nhận lại ;).

    Configuration options

    Bạn cũng có thể thay đổi các cấu hình mặc định của package asset pipeline này. Để làm được việc đó, bạn chạy dòng lệnh sau: 

    $ php artisan config:publish codesleeve/asset-pipeline

    Với việc thi hành lệnh trên, nó sẽ generate ra file cấu hình asset pipeline cho project cụ thể của bạn và nó sẽ tự động ghi đè lên các thiết lập mặc định. Bạn sẽ tìm thấy file cấu hình mà bạn có thể sửa trong thư mục app/config/packages. Bạn hãy xem thêm tài liệu để có những giải thích chi tiết hơn về các tùy chọn cấu hình.

    Hy vọng bạn đã có thể quản lý các asset của mình một cách dễ dàng. Have fun! ;)
