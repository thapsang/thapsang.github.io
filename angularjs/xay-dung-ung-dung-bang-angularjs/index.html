    Xây dựng ứng dụng bằng AngularJS

    Tạo khung ứng dụng AngularJS.
    12 Oct 2014 in Angularjs  9.9 minutes read

    Khi bắt đầu xây dựng một ứng dụng AngularJS, có thể bạn không biết chính xác cách thức tổ chức các files, hoặc thậm chí là không biết mình cần những files nào. Do đó, đội phát triển AngularJS đã tạo ra một dự án mẫu có tên angular-seed, cung cấp cấu trúc khung cơ bản của một ứng dụng AngularJS.

    Nội dung

    • Download khung mẫu
    • Cài đặt công cụ
      • Cài đặt Node.js
      • Cài đặt Grunt CLI
      • Cài đặt Bower
    • Cài đặt phụ thuộc
    • Chạy ứng dụng

    Download khung mẫu

    Để bắt đầu xây dựng các ứng dụng với AngularJS, bạn có thể download hoặc clone ứng dụng mẫu angular-seed trên kho github. Máy tính của bạn cần phải cài đặt git. Bạn có thể tham khảo cách cài đặt git tại đây. Nhập vào Terminal lệnh sau:

    git clone https://github.com/angular/angular-seed.git

    Cài đặt công cụ

    Để chạy được ứng dụng trên local, máy tính của bạn cần cài đặt Node.js, Grunt CLI và Bower.

    Cài đặt Node.js

    Chúng ta cần cài đặt Nodejs để sử dụng NPM - Node Package Management - để tải những thứ mà ứng dụng cần (dependencies) trong quá trình phát triển. Bạn có thể download về phiên bản mới nhất của Node.js tại địa chỉ http://www.nodejs.org/download/ và tiến hành cài đặt nó.

    Nếu bạn sử dụng Linux, bạn có thể cài đặt Node.js bằng command line như sau:

    1 $ sudo apt-get install python-software-properties
2 $ sudo apt-add-repository ppa:chris-lea/node.js
3 $ sudo apt-get update
4 $ apt-get install nodejs

    Tiếp đến, bạn xác nhận nodejs đã được cài đặt thành công hay chưa với lệnh: 

    $ node -v

    Kết quả hiển thị như sau:

    v0.10.32

    Tiếp đến, bạn kiểm tra version của npm với lệnh:

    $ npm -v

    Kết quả hiển thị như sau: 

    1.4.28

    Nếu câu lệnh thực hiện trên máy bạn không trả lại phiên bản của npm, bạn tiến hành cài đặt npm với lệnh sau: 

    $ sudo apt-get install npm

    Cài đặt Grunt CLI

    Grunt CLI là một module của Node.js, nó là một công cụ để tự động hóa các công việc lặp đi lặp lại của việc xây dựng một ứng dụng web bằng dòng lệnh. Chẳng hạn như, mỗi lần bạn lưu lại file Sass hoặc Javascript, Grunt sẽ tự động biên dịch file Sass thành file CSS và tự động kiểm tra và thực hiện các unit test trong kịch bản file Javascript mà bạn không cần tải lại trình duyệt. Có thể hiểu Grunt là một công cụ thực thi các tác vụ javascript.

    Grunt là một NPM package. Để cài đặt Grunt CLI, bạn chạy dòng lệnh sau:

    $ npm install -g grunt-cli

    Cài đặt Bower

    Bower là công cụ quản lý các thành phần phía người dùng (front-end) như html, css, js, etc. Bower liệt kê danh sách các thành phần phụ thuộc trong file bower.json. Trong trường hợp các phụ thuộc của frontend được bằng cách chạy lệnh bower install thì mặc định chúng sẽ được cài đặt vào trong thư mục có tên bower_components.

    Chúng ta cài đặt Bower thông qua NPM tương tự như cách cài đặt Grunt CLI: 

    $ npm install -g bower

    Cài đặt phụ thuộc

    Sau khi cài đặt xong các công cụ NPM, Grunt, Bower, chúng ta cần cài đặt các phụ thuộc của ứng dụng bằng cách chạy lệnh sau:

    $ cd angular-seed
$ npm install

    Với việc thực thi lệnh npm install, nó cũng thực thi luôn lệnh bower install. Chúng ta sẽ tìm thấy 2 thư mục mới trong ứng dụng:

    • node_modules: chứa các npm package là các công cụ chúng ta sử dụng.
    • app/bower_components: chứa các files của khung ứng dụng angularjs.

    Chạy ứng dụng

    Chúng ta tiến hành khởi động máy chủ web bằng cách thực thi lệnh sau:

    $ npm start

    Trên trình duyệt, ứng dụng được chạy tại địa chỉ http://localhost:8000/app/index.html.

    Dưới đây là cấu trúc ban đầu của ứng dụng angular-seed:

    app/                    --> thư mục chứa mã nguồn của ứng dụng
----app.css               --> stylesheet mặc định
----components/           --> thư mục chứa các modules cụ thể của ứng dụng
--------version/              --> component liên quan đến version
------------version.js                 --> khai báo module và giá trị của version
------------version_test.js            --> service test "version"
------------version-directive.js       --> directive dành riêng cho "version"
------------version-directive_test.js  --> version directive tests
------------interpolate-filter.js      --> custom interpolation filter
------------interpolate-filter_test.js --> interpolate filter tests
----view1/                --> the view1 view template and logic
--------view1.html            --> the partial template
--------view1.js              --> the controller logic
--------view1_test.js         --> tests of the controller
----view2/                --> the view2 view template and logic
--------view2.html            --> the partial template
--------view2.js              --> the controller logic
--------view2_test.js         --> tests of the controller
----app.js                --> main application module
----index.html            --> app layout file (the main html template file of the app)
----index-async.html      --> just like index.html, but loads js files asynchronously
karma.conf.js         --> config file for running unit tests with Karma
e2e-tests/            --> end-to-end tests
----protractor-conf.js    --> Protractor config file
----scenarios.js          --> end-to-end scenarios to be run by Protractor

    Có rất nhiều cách để tổ chức cấu trúc thư mục ứng dụng. Chúng ta có thể tổ chức các folder theo kiểu (type) hoặc theo đặc tính (feature). Theo quan điểm của tôi, chúng ta nên tổ chức các folder theo feature. Bạn có thể tham khảo thêm các cách tổ chức cấu trúc thư mục ứng dụng khác nhau như AngularJS Best Practices: Directory Structure hay của Johnpapa để có thể quyết định tổ chức ứng dụng của mình như thế nào cho hợp lý.
