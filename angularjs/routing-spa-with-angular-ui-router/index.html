    Routing ứng dụng SPA sử dụng UI Router module trong AngularJS

    Routing ứng dụng single page sử dụng UI Router thông qua demo.
    04 Nov 2014 in Angularjs  7.76 minutes read

    Phần không thể thiếu trong bất cứ một ứng dụng web nào đó là bộ định tuyến (URL router). Bộ cung cấp định tuyến chuẩn trong AngularJS là ngRoute. Chúng ta đã có bài tìm hiểu về cách routing trong ứng dụng SPA bằng module ngRoute trong bài trước. Trong bài này, chúng ta sẽ sử dụng module UI Router để làm việc đó.

    UI Router là gì?

    AngularUI Router là một khung ứng dụng để định tuyến (routing framework) cho AngularJS, cho phép bạn tổ chức các phần của giao diện vào trong một trạng thái (state). $route service trong module ngRoute tổ chức các phần giao diện theo các routes URL. Không giống như $route service, UI Router tổ chức các phần theo các states. Với một state, bạn có thể có các routes tùy ý cũng như các hành vi khác kèm theo nó.

    Mô hình sử dụng điển hình của UI Router

    States vs. Routes

    Demo

    Trong demo này, chúng ta xây dựng 2 trang chính là Home và Contacts. Trên trang Home, chúng ta sẽ thể hiện đặc tính Multiple Named views và trang Contacts sẽ thể hiện đặc tính Nested views của UI Router.

    Trước hết, để sử dụng UI Router, chúng ta cần gọi đến module ui-router trong file index.html và bơm truyền phụ thuộc cho ứng dụng:

    //index.html
<script src="angular-ui-router.js"></script>

//app.js
angular.module('uiRouterApp', ['ui.router']);

    Trong một file template, chúng ta có thể có một hoặc nhiều vị trí để đặt chỉ thị lệnh ui-view và chúng ta có thể đặt tên hoặc không đặt tên cho các ui-view này. Trong demo, chúng ta sử dụng 1 ui-view directive trong file index.html và 2 ui-sref directive trong thẻ anchor để điều hướng đến state home và contacts.

    <a ui-sref="home">Home</a>
<a ui-sref="contacts">Contact</a>

    ui-sref directive được sử dụng thay thế thuộc tính href của thẻ anchor để tham chiếu đến state. Nếu url tồn tại, nó sẽ trả lại thuộc tính href khi biên dịch. Ví dụ, khi biên dịch 

    <a ui-sref="home">Home</a>

    sẽ được trả lại như sau: 

    <a ui-sref="home" href="#/home">Home</a>

    Trong file app.js, chúng ta sử dụng $stateProvider và $urlRouterProvider service để định nghĩa các states.

    $stateProvider
  .state('home', { url: '/home' })
  .state('contacts', { url: '/contacts' })

    Khi truy cập vào trang Home, chúng ta có một phần nội dung chung và hai nội dung được bơm vào từ bên ngoài. Trong file template partial-home.html, chúng ta thêm 2 view được đặt tên là ui-view="viewA" và ui-view="viewB" (xem chi tiết trong demo). Và với mỗi view, chúng ta sẽ hiển thị các nội dung khác nhau. Khi đó, trong file app.js, chúng ta định nghĩa state home với các views như sau:

    .state('home', {
  url: '/home',
  views: {
    '': { templateUrl: 'partial-home.html'},
    'viewA@home': { template: 'home.viewA' },
    'viewB@home': { 
      templateUrl: 'partial-home-viewB.html',
      controller: function($scope){
        $scope.message = 'This is home.viewB!';
      }
    }
  }
})

    Ở đây, chúng ta sử dụng tên cố định (absolute name) cho view bằng cách khai báo viewA@home và viewB@home để xác định rằng các thiết lập cho hai views này sẽ chỉ ảnh hưởng trong state home. Tài liệu của UI Router giải thích các khái niệm này rất tốt, bạn nên tham khảo nó tại đây.

    Với trang Contacts, chúng ta có hai link Route 1 và Route 2, và chúng ta cũng hiển thị các view tương ứng. Trong trường hợp này, chúng ta sử dụng các view lồng nhau (nested views), và hai link này là các state con của state contacts, do đó, trong khai báo của thẻ anchor trong file partial-contacts.html, chúng ta sử dụng thêm dấu . trong ui-sref directive:

    <a ui-sref=".route1">Route 1</a>
<a ui-sref=".route2">Route 2</a>

    Và chúng ta khai báo state trong ứng dụng để hiển thị các view tương ứng như sau:

    .state('contacts', {
  url: '/contacts',
  templateUrl: 'partial-contacts.html'
})
.state('contacts.route1', {
  url: '/route1',
  template: 'Display content of contacts.route1'
})
.state('contacts.route2', {
  url: '/route2',
  templateUrl: 'partial-contacts-route2.html',
  controller: 'ContactListCtrl'
})

    Trên đây là tổng quan về UI Router trong Angular và cách sử dụng nó để routing trong ứng dụng. Chúng ta sẽ đi vào tìm hiểu chi tiết các đặc tính của module UI Router trong các bài tiếp theo.
